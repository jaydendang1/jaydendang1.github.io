Project Idea - Brawlr
Overview
  • This project will be an app that connects people from local areas to start fights with. A similar app is tinder where others connect to find love or friendship, whereas this app is to connect to others that want to meet up and fight.
  • The concept of this app is for the user to scroll through different fighter's profiles and selecting which fighter they would want to fight. If there is a match between two fighters, the app will connect the two users together where they can communicate, and which will lead them onto the fight.
  • This app will be like the background creation of tinder but cheaper. This project is not a serious project as I believe many will not just have an app to find a local fight. It is a humorous take on social matching apps such as tinder and Grindr.
  • The app will be called Brawlr.
Motivation

    My motivation is to create something more innovative and different from typical social applications. Tinder, Bumble, Grindr all have the same concept of creating friendships or relationships. Whereas, my app will allow people to release frustration and anger on each other, something no other application provides. This could be a cheap way for fighters to train and perfect their skills by fighting many different fighters. There are about 1.5million people using tinder worldwide and 2.4million users active daily on Grindr, this shows how popular matching apps are in society.

Description

    This app will be available for both platforms of IOS apple and Android. Upon opening the app users will be able to create a profile via email, google play account, apple account or social media accounts such as Facebook and Twitter. Furthermore, they will have to input their real details, such as full name, date of birth, address and phone number. They will have to prove this information by submitting a form of identification such as a driver's licence, passport or mail. They will also have to verify their email. Creating a profile will be strict as we don't want any fake accounts or accounts that don't take the app seriously. Upon signing into the account, the app will ask for permission to use location services and access to photos and files. It will then sync the local location and show the user how far away other users are from them that also want to fight. It will then give the user an option to either swipe right or left on another user's profile, meaning when swiping left they have an option to ignore the person or swipe right to agree to fight the user. Upon swiping left or right users will be able to see basic information of the user, such as age, fighting type and body build. Furthermore, if a user swipes right on a user's profile and the user agrees to fight it will allow them to chat to each other. This app will create a community of enthusiastic fighters and bringing people together with the same interests of brawling. It provides the same services as matching apps as it shows users what type of fighters are around, and they can match up with them. This app will available for only Australia first and soon to all the world. There will be in app purchases to have more swipes and free users will be able to have up to 10 swipes a day. There will also be match suggestions depending on the user's profile and the app will provide push notifications when a user has been matched up with.

Tools and Technologies
  • To create this app in a basic form I would be using android app maker which is a free application to create apps without coding. However, it I wanted to advance this app I would have to do the coding myself through java scripting, python and HTML5 to be used as programming languages. Furthermore, we would use AWS Hub as a mobile platform to build and test the app when it is done.
  • Additionally, geolocation is the basis of this application as it connects real users together from certain locations. We would be using google or apple locations maps services to implement geolocation into this app. For android I would apply the classes of Android.Location package and MapView class. For iOS we would be using CLLocationManager Class and Map Kit Framework.
  • For the in app purchases we would be offering users to be able to use paypal or credit card. This is a similar development as how tinder was created.
